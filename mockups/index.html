Challenge 04: Pagination - Detailed Requirements

Backend Requirements

Modify GET /api/products endpoint:

  • Add query parameters: page (default: 1) and limit (default: 10)
  • Return paginated results with metadata
  • Response should include: items, total, page, limit, pages

Modify GET /api/orders endpoint:

  • Add the same pagination parameters
  • Return paginated results with same metadata structure

Example Response:

{
  "items": [
    {
      "id": "product-1",
      "name": "Wireless Headphones",
      "price": 99.99,
      ...
    }
  ],
  "total": 156,
  "page": 1,
  "limit": 10,
  "pages": 16
}

Frontend Requirements

  • Add pagination controls (Previous, Next, page numbers)
  • Update API calls to include pagination parameters
  • Display "Showing X-Y of Z products"
  • "Items per page" selector (bonus)

Implementation Hints

  • Use SQLAlchemy's .offset() and .limit() for database queries
  • Calculate total pages: math.ceil(total / limit)
  • Validate that page number doesn't exceed total pages
  • Return empty array if page is out of bounds (don't error)

Challenge 05: Order Status Management - Detailed Requirements

Backend Requirements

Order Status Enum:

  • Define possible statuses: pending, processing, successful, cancelled, failed
  • Validate status transitions (e.g., can't go from successful to pending)

Update Order Status Endpoint:

  • Create: PATCH /api/orders/{order_id}/status
  • Body: {"status": "processing"}
  • Validate status transition
  • Return updated order

Business Rules:

  • New orders start as pending
  • pending → processing, cancelled
  • processing → successful, failed, cancelled
  • successful → cannot change (terminal state)
  • cancelled → cannot change (terminal state)
  • failed → cannot change (terminal state)

Frontend Requirements

  • Show current status with color-coded badges
  • Add status update buttons (for testing/admin)
  • Filter orders by status (bonus)
  • Display status history (bonus)

Status Colors:

  • pending: Yellow (#fef3c7 / #92400e)
  • processing: Blue (#dbeafe / #1e40af)
  • successful: Green (#d1fae5 / #065f46)
  • cancelled: Gray (#f3f4f6 / #4b5563)
  • failed: Red (#fee2e2 / #991b1b)

Challenge 06: Search & Filtering - Detailed Requirements

Backend Requirements

Search Endpoint:

  • Modify GET /api/products to accept search query parameter
  • Search in: product name, description
  • Case-insensitive search

Filter Parameters:

  • min_price and max_price: Filter by price range
  • sort_by: Sort by price, name, stock
  • order: Sort order asc or desc

Combined Filters:

  • Support multiple filters simultaneously
  • Works with pagination from Challenge 04

Example API Call:

GET /api/products?search=wireless&min_price=20&max_price=150&sort_by=price&order=asc&page=1&limit=10

Frontend Requirements

Search Bar:

  • Real-time search (debounced 300ms)
  • Clear search button
  • Show "No results" message

Filter Panel:

  • Price range slider or inputs
  • Sort dropdown (Price Low-High, High-Low, Name A-Z, etc.)
  • Clear all filters button

Results:

  • Show active filters
  • Display result count
  • Update URL with filter parameters (bonus)

Challenge 07: Shopping Cart Persistence - Detailed Requirements

Current Issue

  • Cart is tied to session_id (browser-based)
  • If user clears cookies, cart is lost
  • Multiple devices don't share cart

Backend Requirements

Improvements:

  • Store cart items in database (already done)
  • Add created_at and updated_at to cart
  • Auto-expire carts after 30 days of inactivity
  • Merge carts when user logs in (if using auth from Challenge 03)

Cleanup Job:

  • Add endpoint to clean expired carts: DELETE /api/carts/cleanup
  • Remove carts older than 30 days with no activity
  • Can be scheduled with cron or called manually

Frontend Requirements

  • Load cart on app start
  • Sync cart when items change
  • Show last updated time
  • Show warning if cart is old (>7 days)
  • Allow user to refresh cart

Acceptance Criteria

  • Cart persists across browser sessions
  • Old carts are automatically cleaned up
  • Cart shows last updated time
  • Cart merges when user logs in (if using auth)
