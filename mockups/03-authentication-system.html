Challenge 03: Authentication System

Implement a complete user authentication system that allows users to register, login, and associate their orders with their account.

üìã Backend Requirements

  • Database Schema: Add a users table with proper relationships to orders
  • Password Security: Hash passwords (use bcrypt, argon2, or similar)
  • Authentication: Implement JWT tokens OR session-based auth
  • User Registration: Validate email uniqueness, password strength
  • User Login: Validate credentials, return auth token/session
  • Protected Endpoints: Secure cart and order endpoints (require authentication)
  • Order Association: Link orders to the authenticated user
  • User Profile: Endpoint to get current user info and their orders

üé® Frontend Requirements

  • Login/Register Forms: Create beautiful, responsive forms (see mockup below)
  • Form Validation: Client-side validation before submission
  • Auth State Management: Store and manage auth token/session
  • Protected Routes: Redirect to login if not authenticated
  • User Header: Show logged-in user name/email in header
  • Logout Functionality: Clear auth state and redirect
  • Persistent Login: Keep user logged in across page refreshes
  • Error Handling: Show meaningful error messages
